Data da revisão: 13/07/2026.

## Critério

Uma entrega somente é classificada como concluída quando o código correspondente está presente na linha principal ou na PR atual, possui teste automatizado aplicável, documentação da decisão e validação pelo CI. Itens descritos apenas como “próximo bloco” não são considerados entregues pela PR original; eles são rastreados até a PR posterior que os implementou.

## Pendências reais verificadas

1. **Explicações por IA operacionalmente incompletas — origem PR 17**
   - criar adaptador HTTP real com saída estruturada;
   - configuração por variáveis de ambiente e secret;
   - endpoint protegido;
   - integração opcional com o dashboard/diagnósticos;
   - timeout, logs sanitizados e testes sem rede.

2. **Cobertura operacional da coleta climática — origem PR 24**
   - teste específico do endpoint `POST /climate/collect`;
   - métricas e logs de recebidos, inseridos, atualizados e inalterados;
   - teste de indisponibilidade e resposta parcial.

3. **Documentação consolidada**
   - atualizar README para refletir o estado atual até a PR 27;
   - registrar variáveis operacionais e sequência de implantação;
   - remover textos antigos que ainda tratam fases já concluídas como futuras.

4. **PR nº 27**
   - somente poderá ser mergeada após CI integral verde e confirmação de que o escopo acima está presente.

## Ordem obrigatória de remediação

1. concluir e mergear a PR nº 27;
2. abrir uma única PR de remediação para os itens 1 a 3 acima;
3. implementar integralmente adaptador de IA, endpoint, integração, cobertura climática e documentação;
4. validar Ruff, Mypy, Pytest e CI;
5. mergear por squash;
6. somente depois retomar novas funcionalidades.

## Regra permanente

Nenhuma PR futura será apresentada como concluída enquanto houver item do próprio escopo marcado como próximo bloco, pendente, TODO ou sem teste aplicável. Uma nova PR só será aberta após a anterior estar integralmente validada e mergeada.
